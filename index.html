Graph Isomorphism

Same structure,
different labels

Before understanding isomorphism, recall that a graph G = (V, E) consists of a non-empty set V of vertices (nodes) and a set E of edges connecting pairs of vertices. Two vertices connected by an edge are called adjacent or neighbours.

Definition — Graph Isomorphism

Two graphs G₁ and G₂ are isomorphic (written G₁ ≅ G₂) if there exists a bijection f: V(G₁) → V(G₂) such that (u,v) ∈ E(G₁) if and only if (f(u),f(v)) ∈ E(G₂). The function f is called the isomorphism.

In plain English: two graphs are isomorphic if one can be relabelled to become the other. They are the same abstract graph — just drawn differently or named differently. Think of two maps of the same city, one in English and one in Hindi. Same roads, same intersections, only names differ — that is isomorphism.

Live Isomorphism Explorer

The Golden Rule — How to Construct a Bijection

You may only map vertex u in G₁ to vertex v in G₂ if deg(u) = deg(v). This is because isomorphism preserves adjacency — if u has 3 neighbours in G₁, then f(u) must also have exactly 3 neighbours in G₂. After grouping by degree, try mappings and verify every edge corresponds.

Check vertex count: |V(G₁)| must equal |V(G₂)|. If different, immediately not isomorphic.

Check edge count: |E(G₁)| must equal |E(G₂)|. Bijection preserves edges.

Compute and compare degree sequences: List degrees of all vertices in each graph, sort both lists. Must be identical.

Construct bijection: Map degree-2 vertices to degree-2, degree-3 to degree-3, etc. Then verify every edge (u,w) in G₁ has a corresponding edge (f(u),f(w)) in G₂.

Critical Warning

The degree sequence check is necessary but not sufficient. Two graphs can have identical vertex count, edge count, and degree sequence — and still not be isomorphic. The only proof of isomorphism is exhibiting a valid bijection. The only proof of non-isomorphism is either a failed invariant OR proving no valid bijection exists.

Q1: G₁ degrees=[1,2,2,3], G₂ degrees=[2,2,2,2]. Are they isomorphic?▼
No. Degree sequences differ: [1,2,2,3] ≠ [2,2,2,2]. G₁ has a degree-1 vertex and a degree-3 vertex which G₂ lacks. Since degree sequence is a graph invariant preserved under isomorphism, these graphs cannot be isomorphic. No need to attempt a bijection.
Q2: G₁: 4-cycle (a-b-c-d-a). G₂: 4-cycle (1-2-3-4-1). Are they isomorphic?▼
|V|=4 ✓, |E|=4 ✓, degree sequence=[2,2,2,2] for both ✓. Bijection: f(a)=1,f(b)=2,f(c)=3,f(d)=4. Check: (a,b)→(1,2)✓, (b,c)→(2,3)✓, (c,d)→(3,4)✓, (d,a)→(4,1)✓. Isomorphic.
Q3: Why is graph isomorphism considered a hard computational problem?▼
Graph Isomorphism has no known polynomial-time algorithm for general graphs, but is also not proven to be NP-complete — it sits in its own complexity class (GI). For special cases (trees, planar graphs, bounded degree) polynomial algorithms exist. This makes it one of few problems in a "grey zone" of complexity theory, and remains an active research area.

Paths & Cycles

Walking through a graph

When we move through a graph following edges, we create different kinds of sequences depending on what repetitions we allow. These form the foundation for Eulerian paths, Hamiltonian paths, connectivity analysis, and almost everything else in graph theory.

Definition — Walk

A walk is a sequence of vertices v₁,v₂,...,vₙ where each consecutive pair (vᵢ,vᵢ₊₁) is connected by an edge. Both vertices and edges can repeat. Length = number of edges traversed (counting repetitions).

Definition — Trail

A trail is a walk where no edge repeats, but vertices may repeat. Think of it as walking through a city never using the same road twice — but revisiting junctions is fine.

Definition — Path (Simple Path)

A path is a walk where neither vertices nor edges repeat. When Rosen says "path" without qualification, this is what he means. Length = number of edges = (number of vertices) − 1.

Definition — Circuit

A circuit is a closed trail — starts and ends at the same vertex, no edge repeats, but intermediate vertices may repeat.

Definition — Cycle (Simple Cycle)

A cycle is a closed path — starts and ends at the same vertex, no edge repeats AND no intermediate vertex repeats. Cycle of length k = k edges, k distinct vertices. Smallest possible cycle: length 3 (triangle, C₃).

Walk → Trail → Path → Cycle — Click to Explore

Circuit vs Cycle — The Exact Distinction

Both are closed (start = end) and both have no repeated edges. The only difference:

A circuit allows intermediate vertex repetition — you can pass through the same junction multiple times, using different roads each time. A cycle does NOT allow intermediate vertex repetition — every junction visited exactly once.

Hierarchy: Cycle ⊂ Circuit ⊂ Trail ⊂ Walk. Every cycle is a circuit, but not every circuit is a cycle.

Theorem — Connectivity and Paths

A graph G is connected if and only if there exists a path between every pair of vertices. This is the fundamental definition of connectivity — no pair of vertices is stranded without a route between them.

Theorem — Long Walks Contain Cycles

Any walk of length ≥ n in an n-vertex graph must repeat a vertex, hence contains a cycle. A path can visit at most n distinct vertices, so length n−1 at most. Any longer walk must revisit.

Q1: List all simple paths from A to D in V={A,B,C,D}, E={(A,B),(B,C),(C,D),(A,C),(B,D)}.▼
Simple paths (no repeated vertices):
• A→B→D (length 2)
• A→B→C→D (length 3)
• A→C→D (length 2)
• A→C→B→D (length 3)
All four are valid. Note the lengths — 2 edges and 3 edges.
Q2: Is a walk that visits every edge of a graph always an Eulerian circuit?▼
No. An Eulerian circuit is a circuit (closed trail) that uses every edge exactly once. A walk might use edges multiple times. For Eulerian we specifically need a circuit (closed, no edge repeats) covering all edges — not just any walk.

Graph Colouring

Minimum colours,
no two neighbours match

Graph colouring arises naturally in scheduling (no two conflicting tasks share a time slot), register allocation in compilers, map colouring, and frequency assignment in wireless networks. The constraint is always the same: adjacent vertices must get different colours.

Definition — Proper Vertex Colouring

An assignment of colours to vertices of G such that no two adjacent vertices receive the same colour. Colours are just labels — numbers, names, or actual colours.

Definition — Chromatic Number χ(G)

The chromatic number χ(G) ("chi of G") is the minimum number of colours needed to properly colour all vertices of G. Finding χ(G) for a general graph is NP-hard.

Chromatic Number Explorer — See Optimal Colourings

Why Odd Cycles Force a Third Colour

This is the single most important intuition in graph colouring. Consider a triangle A–B–C (3-cycle). Try 2 colours: A=Purple → B=Teal (adjacent to A) → C is adjacent to both A(Purple) and B(Teal). C cannot be either → third colour forced.

For a square A–B–C–D (4-cycle, even): A=Purple, B=Teal, C=Purple, D=Teal. D is adjacent to C(Purple) and A(Purple). D=Teal works. Only 2 colours. Even cycles need 2, odd cycles need 3. This is why odd cycles and bipartite graphs are so connected.

Theorem — Greedy Upper Bound

χ(G) ≤ Δ(G) + 1 where Δ(G) is the maximum degree.

Why it works: Colour vertices one by one. When you reach vertex v, it has at most Δ(G) already-coloured neighbours, each potentially using a different colour. At most Δ(G) colours are forbidden. Colour Δ(G)+1 is always available.

Theorem — Brook's Theorem (1941)

For any connected G that is neither Kₙ nor an odd cycle: χ(G) ≤ Δ(G).

This tightens the greedy bound by 1 for most graphs. The exceptions are Kₙ (where χ=n=Δ+1) and odd cycles (where χ=3 but Δ=2, so Δ+1=3 is tight). For all other connected graphs, one fewer colour than the greedy bound suffices.

Theorem — Clique Lower Bound

If G contains Kₙ as a subgraph (a clique of size n): χ(G) ≥ n.

In a clique, all n vertices are mutually adjacent — each needs a distinct colour. The chromatic number of the entire graph must be at least as large as the largest clique.

Theorem — Four Colour Theorem (1976)

Every planar graph can be properly coloured with at most 4 colours.

Open problem for 100+ years. Proved by Appel and Haken in 1976 using a computer to check 1,936 cases — the first major theorem proved with computer assistance. Means any political map needs at most 4 colours.

Q1: Find χ(K₅) and justify.▼
K₅ has 5 vertices, all mutually adjacent. Every vertex needs a colour different from all 4 others. χ(K₅)=5. Also Δ(K₅)=4 and Kₙ is the exception in Brook's theorem, so χ=Δ+1=5. ✓
Q2: G has 6 vertices each of degree 4. What is the upper bound on χ(G)?▼
Δ(G)=4. Greedy bound: χ ≤ 5. G is not K₅ (would need 5 vertices all degree 4, but K₅ has degree 4 — actually G could be K₅ with an extra isolated vertex, but here all 6 have degree 4). If G is not Kₙ or an odd cycle, Brook's theorem gives χ(G) ≤ 4.

Critical Path Method

The longest path = your deadline

Critical Path Method (CPM) is a project scheduling technique using weighted directed graphs. It finds the minimum possible project duration by identifying the longest chain of dependent tasks — the bottleneck chain you cannot speed up without doing the tasks faster.

Definition — DAG (Directed Acyclic Graph)

A directed graph with no directed cycles. Real projects use DAGs because no task can depend on itself (circular dependencies don't exist in practice). Vertices = events/tasks, directed edges = dependencies ("A must finish before B starts"), edge weights = task durations.

Definition — Critical Path

The longest weighted directed path from source (project start) to sink (project end) in the DAG. Its total weight = minimum possible project duration. Any task on the critical path has zero float — delaying it directly delays the project.

EST and LST — The Two Passes

EST (Earliest Start Time) = earliest time a task can begin, assuming all predecessors start as early as possible. Computed by forward pass — go left to right, take the maximum over all incoming paths. We take maximum because the task cannot start until ALL predecessors finish, and the last to finish determines the earliest start.

LST (Latest Start Time) = latest time a task can begin without delaying the project. Computed by backward pass — go right to left, take the minimum over all outgoing paths. We take minimum because starting later than any successor requires will delay that successor.

Float (Slack) = LST − EST. How much a task can be delayed without affecting the deadline. Float=0 means critical.

Forward Pass → Backward Pass → Float → Critical Path

Float Formula — Covers Every Exam Scenario

Q1: Why is the critical path the LONGEST path, not shortest?▼
Tasks on the critical path are sequentially dependent — each must wait for the previous. The project can't finish until ALL chains complete — the longest chain finishes last and determines the deadline. Shortest path through the DAG would be faster but doesn't represent the bottleneck. You can't skip the longer chains.
Q2: Task B has float=3. Its duration increases by 5. How much is the project delayed?▼
Delay = max(0, k − float) = max(0, 5−3) = 2 days. First 3 days absorb into float. Remaining 2 delay the project. B's new float = 0 (becomes critical).
Q3: Can there be multiple critical paths? What does it imply?▼
Yes — if two or more paths from start to finish have equal maximum total length, all are critical. This makes the project more fragile — any critical task on any critical path delays everything. More tasks have zero float, less scheduling flexibility.

Eulerian Paths & Circuits

Traverse every edge exactly once

In 1736, Euler solved the Königsberg Bridge Problem — can you walk through Königsberg crossing every bridge exactly once? He proved it impossible and gave the reason in terms of vertex degrees. This invented graph theory.

Definition — Eulerian Path

A trail (no edge repeated) that visits every edge exactly once. Vertices may be revisited. May start and end at different vertices.

Definition — Eulerian Circuit

An Eulerian path that starts and ends at the same vertex. Traverses every edge exactly once and returns to start.

Key distinction: Eulerian is about edges. Every edge used exactly once. Vertices can repeat. Different from Hamiltonian (Topic 6) which is about vertices.

Euler's Insight — Why Degree is Everything

Every time you enter a vertex via some edge, you must exit via a different edge (except at start/end of journey). This means most vertices must have their edges "paired" — one in, one out — requiring even degree. Only the start and end vertices can have unpaired edges (odd degree). So exactly 0 vertices (circuit) or 2 vertices (path) may have odd degree.

Theorem — Eulerian Circuit (Euler, 1736)

A connected graph G has an Eulerian circuit iff every vertex has even degree.

In a circuit, you enter and exit every vertex equally — each visit uses 2 edges (one in, one out), so degree must be even. For digraphs: every vertex must have deg⁻(v) = deg⁺(v).

Theorem — Eulerian Path

A connected graph G has an Eulerian path (not circuit) iff it has exactly 2 odd-degree vertices.

The path has a distinct start and end. Start: you leave one more time than arrive → odd degree. End: you arrive one more time than leave → odd degree. All intermediate vertices: equal arrivals and departures → even degree. Path runs from one odd-degree vertex to the other.

Why 1 or 3 Odd-Degree Vertices is Impossible

By the Handshaking Lemma, Σdeg = 2|E| = even. Sum of even-degree vertices = even. So sum of odd-degree contributions = even. A sum of odd numbers is even only if there are an even count of them. So odd-degree vertices always number 0, 2, 4, 6, ... — never 1, 3, 5 etc.

Hierholzer's Algorithm — Finding the Eulerian Circuit

Start at any vertex. Traverse unused edges until you return to start → this gives a sub-circuit.

Find any vertex on the sub-circuit with unused edges. Start a new sub-circuit from there.

Merge the new sub-circuit into the existing one at the shared vertex.

Repeat until all edges used. The merged result is the complete Eulerian circuit.

Q1: V={A,B,C,D,E}, E={(A,B),(A,D),(B,C),(B,D),(C,E),(D,E)}. Eulerian circuit or path?▼
Degrees: A=2, B=3, C=2, D=3, E=2. Odd-degree: B and D (exactly 2). Connected ✓. Eulerian path from B to D (or D to B). No circuit since B and D are odd.
Q2: Graph has 8 vertices with degrees 2,2,4,4,2,2,3,3. Eulerian circuit?▼
Odd-degree vertices: the two degree-3 vertices → exactly 2 odd. No Eulerian circuit (needs all even). Eulerian path exists between the two degree-3 vertices (if connected).

Hamiltonian Paths & Circuits

Visit every vertex exactly once

Hamiltonian paths look similar to Eulerian on the surface — both traverse a graph in one go. But the target is completely different, and the difficulty is incomparably harder.

Definition — Hamiltonian Path

A path that visits every vertex of the graph exactly once. You don't need to use every edge — just every vertex, once each.

Definition — Hamiltonian Circuit

A Hamiltonian path that also returns to the starting vertex — visiting every vertex exactly once and completing a cycle. Also called Hamiltonian cycle.

Sufficient Conditions — When Existence is Guaranteed

Since no simple necessary-and-sufficient condition is known, we use sufficient conditions — they guarantee existence when satisfied, but failure doesn't mean non-existence.

Theorem — Dirac's Theorem (1952)

If G has n ≥ 3 vertices and every vertex has degree ≥ n/2, then G has a Hamiltonian circuit.

Intuition: If every vertex is connected to at least half the others, the graph is dense enough that you can always find a way to visit all vertices in a cycle. With fewer connections you might get stuck with no unvisited vertex reachable.

Example: n=6 → need degree ≥ 3. Every vertex degree ≥ 3 → Hamiltonian circuit guaranteed.

Theorem — Ore's Theorem (1960)

If G has n ≥ 3 vertices and for every pair of non-adjacent vertices u,v: deg(u)+deg(v) ≥ n, then G has a Hamiltonian circuit.

Dirac is a special case of Ore — if every vertex has degree ≥ n/2, then any non-adjacent pair satisfies deg(u)+deg(v) ≥ n/2+n/2=n. Ore's theorem is slightly more general — it only requires the condition for non-adjacent pairs.

Necessary Conditions for NON-Existence

Condition 1: Any vertex with degree 1 makes Hamiltonian circuit impossible — you need to both enter and exit, requiring ≥ 2 edges.

Condition 2 (Cut test): If removing k vertices creates more than k+1 components, no Hamiltonian circuit exists. In a Hamiltonian circuit, removing k vertices splits it into at most k pieces (at most k+1 if split at endpoints). More than k+1 components means the circuit cannot exist.

Condition 3: Every vertex uses exactly 2 of its edges in a Hamiltonian circuit. If a vertex has degree 2, both its edges MUST be in the circuit — this forces local structure and can create contradictions.

Q1: G has n=8 vertices, every vertex has degree 5. Does a Hamiltonian circuit exist?▼
n/2 = 4. Every vertex has degree 5 ≥ 4. Dirac's theorem applies → Hamiltonian circuit guaranteed to exist. No need to find it — the theorem guarantees existence.
Q2: Removing vertex X creates 4 components. Can G have a Hamiltonian circuit?▼
Removed k=1 vertex, got 4 components. For Hamiltonian circuit: removing k vertices creates at most k+1=2 components. But 4 > 2. No Hamiltonian circuit exists.
Q3: Why is Hamiltonian circuit detection harder than Eulerian?▼
Eulerian detection requires only checking vertex degrees — O(V). Hamiltonian detection is NP-complete — no polynomial algorithm is known for general graphs. Best algorithms are exponential worst case. Connected to P vs NP — an efficient Hamiltonian algorithm would prove P=NP, solving one of the greatest open problems in computer science.

Bipartite Graphs

Two teams — all edges between teams

Bipartite graphs model two-sided relationships: students and courses, workers and jobs, users and products. In all these cases the two "sides" of the relationship form a natural bipartition — connections exist between sides, never within a side.

Definition — Bipartite Graph

G=(V,E) is bipartite if V can be partitioned into two disjoint non-empty sets X and Y (the bipartition) such that every edge has one endpoint in X and one in Y. No edge connects two vertices within X, and none within Y.

Definition — Complete Bipartite K(m,n)

K(m,n) has |X|=m, |Y|=n, and every vertex in X connects to every vertex in Y. Total edges = m×n. deg(X vertices)=n, deg(Y vertices)=m.

Definition — Star K(1,n)

K(1,n) has 1 centre vertex connected to n leaf vertices. All trees are bipartite — stars are the simplest case.

Bipartite Graph Explorer
Fundamental Theorem — Bipartite ↔ No Odd Cycles

A graph G is bipartite if and only if it contains no odd-length cycle.

Forward (bipartite → no odd cycles): Every edge alternates X→Y→X→Y... Completing a cycle requires returning to the starting side — must take an even number of steps. Odd cycles impossible.

Backward (no odd cycles → bipartite): Do BFS from any vertex. Colour vertices at even BFS distance Purple (set X), odd distance Teal (set Y). Since no odd cycles exist, no edge connects two vertices of the same colour. Valid bipartition found.

Connection to colouring: Bipartite → χ(G)=2. Odd cycle → χ≥3. This is exactly the same condition.

BFS 2-Colouring Algorithm — Detecting Bipartiteness

Pick any vertex, colour it Purple. Add to queue.

For each vertex dequeued: assign opposite colour to each uncoloured neighbour. Add them to queue.

Conflict check: If any neighbour already has the same colour as current → NOT bipartite (odd cycle found).

BFS completes without conflict → bipartite. Purple = set X, Teal = set Y.

Q1: Is C₈ bipartite? What about C₇?▼
C₈ is even (8 edges) → bipartite. No odd cycles. Bipartition: {1,3,5,7} and {2,4,6,8}.
C₇ is odd (7 edges) → NOT bipartite. It IS itself an odd cycle.
Q2: Prove every tree is bipartite.▼
Proof: A tree is acyclic — it has no cycles at all, hence no odd cycles. By the fundamental theorem (bipartite ↔ no odd cycles), every tree is bipartite. ✓ The bipartition is found by BFS from root, colouring alternately level by level — no cycles means no conflict ever arises.
Q3: K(3,3) — how many edges? Why is it famous?▼
3×3 = 9 edges. Famous because K(3,3) is one of the two forbidden minors in Kuratowski's theorem — it is non-planar (cannot be drawn without crossing). It represents the unsolvable "3 houses 3 utilities" puzzle.

Digraphs (Directed Graphs)

Direction changes everything

In all previous topics, edges were undirected — connecting u and v means travel both ways. In a digraph, each edge has a direction. One-way streets, social media follows, course prerequisites — all naturally modelled as directed edges.

Definition — Digraph

G=(V,E) where E consists of directed edges (arcs). Each arc is an ordered pair (u,v) — goes FROM u TO v. Arc (u,v) and arc (v,u) are different edges — unlike undirected graphs.

Definition — In-degree and Out-degree

deg⁻(v) = number of arcs pointing INTO v. deg⁺(v) = number of arcs going OUT from v. Vertex with deg⁺=0 is a sink. Vertex with deg⁻=0 is a source.

Theorem — Handshaking for Digraphs

Σ deg⁻(v) = Σ deg⁺(v) = |E|.

Every arc (u,v) contributes 1 to out-degree of u and 1 to in-degree of v. Both sums count every arc once. Both equal |E|.

Strong vs Weak Connectivity and SCCs
Definition — Strongly Connected

A digraph is strongly connected if for every ordered pair (u,v), a directed path from u to v AND from v to u exists. You can reach any vertex from any other following arrow directions.

Definition — Weakly Connected

A digraph is weakly connected if the underlying undirected graph (ignoring directions) is connected. But following arrows you might not get from every vertex to every other.

Definition — Strongly Connected Component (SCC)

A maximal subgraph that is strongly connected. Every digraph decomposes uniquely into its SCCs. Collapsing each SCC to one node gives a DAG.

Adjacency Matrix and Its Powers

For a digraph with n vertices, the adjacency matrix A is n×n where A[i][j]=1 if arc (i,j) exists. Row sum = out-degree; column sum = in-degree. Matrix is not necessarily symmetric.

Aᵏ[i][j] = number of directed walks of length exactly k from vertex i to vertex j, computed by matrix multiplication. Very commonly asked in exams.

Eulerian Conditions for Digraphs

Circuit: Every vertex v must have deg⁻(v) = deg⁺(v).
Path: Exactly one vertex with deg⁺−deg⁻=1 (start), one with deg⁻−deg⁺=1 (end), all others balanced.

Q1: Arcs: (1,2),(2,3),(3,1),(1,4),(4,2). Eulerian circuit or path?▼
Out: 1→2, 2→1, 3→1, 4→1. In: 1→1, 2→2, 3→1, 4→1. Vertex 1: out=2,in=1 ≠. Vertex 2: out=1,in=2 ≠. No Eulerian circuit. Vertex 1 has out−in=1 (start), vertex 2 has in−out=1 (end). Others balanced → Eulerian path from 1 to 2.
Q2: What does A³[i][j] mean in an adjacency matrix?▼
A³[i][j] = number of directed walks of length exactly 3 from vertex i to vertex j. Computed as A×A×A by standard matrix multiplication. Aᵏ[i][j] generalises this to walks of length k.

Multigraphs & Pseudographs

Beyond simple graphs

All previous topics assumed a simple graph — at most one edge between any pair, no self-loops. Real situations often violate this: multiple flight routes between two cities, or a process with a feedback loop to itself. These require multigraphs and pseudographs.

Definition — Simple Graph

At most one edge between any pair of vertices and no self-loops. Standard assumption throughout Topics 1–8.

Definition — Multigraph

Allows multiple parallel edges between the same pair of vertices. No self-loops. The number of edges between u and v is the multiplicity of that edge. Königsberg bridge problem is a multigraph.

Definition — Pseudograph

Allows parallel edges AND self-loops. A self-loop is edge (v,v) — starts and ends at same vertex. Contributes 2 to deg(v) (one for each endpoint, both are v).

Simple vs Multigraph vs Pseudograph

How Previous Theorems Extend

Handshaking Lemma: Still holds. Σdeg(v)=2|E| where |E| counts each parallel edge separately and each self-loop contributes 2 to degree.

Eulerian circuits: Same degree conditions apply. Connected multigraph has Eulerian circuit iff all degrees even.

Graph colouring: Parallel edges don't change colouring — adjacent vertices still just need different colours. But a self-loop makes proper colouring impossible — vertex would need a colour different from itself.

Graph isomorphism: Bijection must also preserve edge multiplicities — if u-v has multiplicity 3 in G₁, then f(u)-f(v) must also have multiplicity 3 in G₂.

Q2: Why does a self-loop contribute 2 to degree, not 1?▼
An edge contributes 1 to each of its two endpoints. A self-loop (v,v) has both endpoints = v. So it contributes 1+1=2 to deg(v). This ensures Handshaking Lemma Σdeg=2|E| still holds — each edge (including self-loops) contributes exactly 2 to the total.

Handshaking Lemma

The most fundamental identity in graph theory

The Handshaking Lemma gets its name from a social analogy: if n people shake hands at a party, the total recorded per person equals twice the actual handshakes — each handshake counted once by each participant. In graph terms, each edge is counted once by each of its two endpoints.

The Handshaking Lemma

For any undirected graph G=(V,E): Σᵥ∈V deg(v) = 2|E|. The sum of degrees of all vertices equals twice the number of edges.

Σ deg(v) = 2|E|

Proof

Consider any edge e = {u,v}. When computing deg(u), edge e contributes 1. When computing deg(v), edge e contributes 1 again. So each edge contributes exactly 2 to the total sum Σdeg(v). Since there are |E| edges, total = 2|E|. This holds for every graph, regardless of structure. ✓

Corollary — Number of Odd-Degree Vertices is Always Even

In any graph, the number of vertices with odd degree is always even (0, 2, 4, 6, ...).

Proof: Let V_odd = set of odd-degree vertices, V_even = set of even-degree vertices.
Σdeg = 2|E| = even (total).
Σ_{V_even} deg = sum of evens = even.
∴ Σ_{V_odd} deg = even − even = even.
Each term in Σ_{V_odd} deg is odd. Sum of k odd numbers is even iff k is even.
∴ |V_odd| is even. ✓

Consequence: A graph can have 0, 2, 4, 6, ... odd-degree vertices — never 1, 3, 5 etc. This is why Eulerian conditions only consider 0 or 2 odd-degree vertices — those are the only even possibilities ≤ 2.

Common Uses in Exams

Find unknown degree: Given all degrees but one, use Σdeg=2|E| (the sum must be even, so find the missing value and its parity).

Find edge count: Given all degrees, sum them and divide by 2 to get |E|.

Find vertex count: In a d-regular graph (all vertices degree d), n = 2|E| / d.

Prove impossibility: If a proposed graph requires an odd number of odd-degree vertices, it cannot exist.

Q1: Degrees 1,2,3,4,2,3,d. Find d and |E|.▼
Sum = 15+d must be even → d is odd. Minimum d=1. With d=1: Σdeg=16, so |E|=8. d can be any odd value ≥ 1 within constraints.
Q2: Can a graph have exactly 5 odd-degree vertices? Prove.▼
No. By the corollary, count of odd-degree vertices must be even. 5 is odd → impossible. Proof: Σdeg=2|E|=even. Σ_{even-deg}=even. Σ_{odd-deg}=even−even=even. Sum of k odd numbers = even iff k is even. ∴ k must be even. 5 is not even. ✓
Q3: 4-regular graph with 10 edges. How many vertices?▼
Σdeg = 2×10 = 20. Each vertex contributes 4. Number of vertices = 20÷4 = 5. Verify: 5×4=20=2×10 ✓
Q4: Full proof — number of odd-degree vertices is even.▼
Let G=(V,E). Partition V into V_even and V_odd.
By Handshaking: Σ_{v∈V} deg(v) = 2|E| → even.
Σ_{V_even} deg(v) = sum of even numbers = even.
Therefore: Σ_{V_odd} deg(v) = 2|E| − Σ_{V_even} deg(v) = even − even = even.
Each term in Σ_{V_odd} deg(v) is odd. Sum of an odd number of odd terms is odd; sum of an even number is even. Since sum = even → number of terms = |V_odd| is even. ✓

Dijkstra's Algorithm

Shortest path in a weighted graph

Published by Edsger W. Dijkstra in 1959, this algorithm solves the single-source shortest path problem: given a weighted graph and source vertex, find the shortest (minimum total weight) path to every other vertex. It powers GPS navigation, network routing protocols, and game AI pathfinding.

Problem Statement

Given weighted graph G=(V,E,w) with w(e)≥0 for all edges, and source vertex s∈V — find the minimum-weight path from s to every v∈V.

Critical Constraint — Non-Negative Weights Only

Dijkstra requires all edge weights ≥ 0. With negative weights it gives wrong answers. Use Bellman-Ford for graphs with negative weights — it is slower but correct, and can detect negative cycles.

The Core Idea — Greedy Expansion

Dijkstra is a greedy algorithm. At each step, it picks the unvisited vertex with the currently smallest known distance from source, "locks in" that distance as final, then updates distances of its neighbours. This works because all weights are non-negative: once a vertex u is selected as minimum among unvisited, no future path through another unvisited vertex (with distance ≥ dist(u)) plus non-negative edge weight can give a shorter path to u.

Algorithm Steps — Detailed

Initialise: dist(source)=0, dist(v)=∞ for all others. Mark all unvisited.

Select current: Pick unvisited vertex u with minimum dist(u). First iteration picks source.

Relax neighbours: For each unvisited neighbour v: compute candidate = dist(u)+w(u,v). If candidate < dist(v) → update dist(v)=candidate, record prev(v)=u.

Mark u visited. Its distance is now final — never updated again.

Repeat steps 2–4 until all vertices visited or all remaining have dist=∞ (unreachable).

Time: O(V²) with array  |  O((V+E) log V) with priority queue

After algorithm completes, trace the "prev" (Via) column backwards: To find path S→T: T←C←B←S. So path = S→B→C→T, cost = 2+3+4 = 9.

Init: A=0,B=∞,C=∞,D=∞.
Process A: B=1, C=4.

Process C(3): D=min(7,3+3)=6.
Process D(6): done.
Shortest A→D = 6 via A→B→C→D (1+2+3).
Q2: Why does Dijkstra fail on negative weights?▼
Dijkstra finalizes a vertex's distance when first selected. With negative edges, a later path might be shorter. Example: S→A(3), S→B(4), B→A(−2). Dijkstra finalizes A=3. But path S→B→A=4+(−2)=2 is shorter! Dijkstra already finalized A=3 and won't reconsider. → Wrong answer. Bellman-Ford handles this with V−1 full passes allowing updates even after processing.
Q3: After Dijkstra, vertex X shows ∞. What does this mean?▼
Vertex X is unreachable from source S — no path exists. In undirected graph: X is in a different connected component. In digraph: no directed path from S to X exists (even if X→S path exists).
Q4: Prove that once Dijkstra visits a vertex, its distance is final.▼
Proof by contradiction: Suppose vertex u is selected with dist(u)=d, and a shorter path exists with cost d'<d. That path must pass through some unvisited vertex w. But dist(w)≥d (since u was minimum among unvisited). Path from w to u adds ≥0 (non-negative weights). So d'≥dist(w)≥d. Contradiction with d'<d. ∴ dist(u) is final when selected. ✓
